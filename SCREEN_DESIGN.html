Section 4

데이터 흐름 & 실시간 업데이트

1. 초기 로딩

useEffect → fetchData()

GET /api/overview?days=7
GET /api/campaigns
GET /api/performance/timeline
GET /api/alerts?limit=10

Promise.all 병렬 호출

2. WebSocket 연결

new WebSocket(WS_URL)

onopen → setConnected(true)
onmessage → type 분기
performance_update → fetchData()
creative_* → fetchContentData()

서버 데이터 수집 시 자동 갱신

3. 자동 폴링

setInterval 60초

60초마다 fetchData()
기간 변경 시 즉시 재호출
cleanup: clearInterval

WebSocket 끊김 시 폴백

4. 채팅 인터랙션

handleSend()

POST /api/chat { message }
AdManagerSkill 처리
결과 → chatHistory 추가
scrollIntoView 자동 스크롤

한국어/영어 NL 커맨드

Section 6

반응형 & 인터랙션

사이드바 토글

sidebarOpen 상태로 w-64 ↔ w-0 전환. transition-all duration-300 애니메이션. 탑바 좌측 햄버거 버튼으로 토글.

WebSocket 실시간

performance_update 이벤트 → 전체 데이터 재호출. creative/ab_test/audience 이벤트 → Content 데이터만 재호출. 연결 끊김 시 60초 폴링 폴백.

에러 처리

ErrorBoundary로 전체 크래시 방지. 재로드 버튼 제공. API 에러 시 catch → console.error, 채팅 에러 시 "서버 연결 오류" 메시지.
