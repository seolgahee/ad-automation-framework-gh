사전 준비 — Node.js 설치
이 프레임워크를 실행하려면 Node.js v22 이상이 필요합니다.

내 운영체제 선택:

가장 쉬운 방법: nodejs.org 접속 → LTS 버전 (v22+) macOS Installer 다운로드 → .pkg 파일 실행 → "다음" 버튼만 누르면 끝!
또는 Homebrew로 설치
brew install node@22
설치 방법: nodejs.org 접속 → LTS 버전 Windows Installer (.msi) 다운로드 → 실행 → "Next" 계속 클릭. "Add to PATH" 체크 확인!
⚠ 중요: 설치 후 반드시 명령 프롬프트(cmd)를 새로 열어서 확인하세요.
Ubuntu / Debian
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs

✅ 설치 확인

Terminal / 명령 프롬프트
$ node -v
v22.x.x  ← 이렇게 나오면 성공!
Node.js v22 이상이 설치되었습니다

API 키 연결 (가장 중요!)
광고 플랫폼과 연결하기 위한 인증 키를 설정합니다. 아래에 입력하면 .env 파일을 자동 생성해드립니다.
🔑 API 키란? 광고 플랫폼에서 발급하는 "비밀번호" 같은 것입니다. 최소 1개 플랫폼의 키만 있으면 시작할 수 있어요.
Meta API 키 발급:
1
developers.facebook.com 로그인 → "내 앱" → "앱 만들기"
2
앱 대시보드에서 앱 ID와 앱 시크릿 확인
3
Graph API Explorer에서 액세스 토큰 발급 (ads_management, ads_read 권한 체크)
4
광고 관리자 URL에서 광고 계정 ID 확인 (act_로 시작)
Google Ads API 키 발급:
1
Google Ads → 도구 → API 센터에서 개발자 토큰 신청
2
Cloud Console → OAuth 2.0 클라이언트 만들기
3
OAuth Playground에서 Refresh Token 발급
4
Google Ads 상단의 고객 ID (XXX-XXX-XXXX 형식)
TikTok API 키 발급:
1
business-api.tiktok.com 로그인 → 앱 생성
2
Marketing API 승인 후 Access Token 복사
3
Ads Manager에서 Advertiser ID 확인
선택사항입니다. 나중에 추가해도 됩니다. 설정하면 ROAS 하락 등의 알림을 실시간으로 받습니다.
기본값 그대로 써도 됩니다. 특별히 변경할 이유가 없다면 건너뛰세요.
.env 파일 내용 

      

        📄 저장 방법: ① "전체 복사" 클릭 → ② 메모장/텍스트편집기에 붙여넣기 → ③ 프로젝트 폴더에 .env 이름으로 저장

        ※ Windows: 저장 시 "파일 형식"을 "모든 파일"로 변경 후 이름에 .env 입력

📊 대시보드 주요 기능

실시간 KPI

🔌 REST API 사용법 (고급)

    
대시보드 외에도 API를 직접 호출할 수 있습니다. 개발자용이므로 일반 사용자는 건너뛰어도 됩니다.


    

      
      

        
$ curl http://localhost:3099/api/overview?days=7

        
기간별(1/7/14/30일) 총 지출, 전환수, ROAS, CPA, CTR, 플랫폼별 비교 데이터를 반환합니다.

      

    

    

      
      

        
$ curl http://localhost:3099/api/campaigns

        
모든 캠페인의 이름, 상태, 예산, 오늘 지출/전환, 주간 ROAS 등을 반환합니다.

      

    

    

      
      

        
$ curl -X POST http://localhost:3099/api/campaigns/1/budget \
  -H "Content-Type: application/json" \
  -d '{"daily_budget": 100000}'

        
특정 캠페인의 일일 예산을 변경합니다. WebSocket으로 실시간 반영됩니다.

      

    

    

      
      

        
$ curl -X POST http://localhost:3099/api/campaigns/1/status \
  -H "Content-Type: application/json" \
  -d '{"status": "PAUSED"}'

        
가능한 상태: ACTIVE, PAUSED, ENABLED, DRAFT

?

    
자주 묻는 질문

  


  
node -v로 v22 이상인지 확인 → 프로젝트 폴더 안인지 확인 → rm -rf node_modules && npm install 재시도

  
npm run dev 터미널이 열려있는지 확인 → 포트 충돌 시 .env에서 DASHBOARD_PORT를 3200으로 변경 → http:// 포함해서 주소 입력

  
.env 파일을 텍스트 편집기로 열어 수정 → 서버 재시작 (Ctrl+C 후 npm run dev)

  
사용하려는 플랫폼 키만 .env에 입력하면 됩니다. 나머지는 비워두세요. 프레임워크가 자동으로 인식합니다.

  
Mac: Finder에서 Cmd+Shift+. 누르기 / Windows: 파일 탐색기 → 보기 → "숨긴 항목" 체크

  
기본 15분 간격으로 자동 수집됩니다. .env의 COLLECT_INTERVAL_MINUTES 값을 변경하면 주기를 조절할 수 있습니다. 수동 수집은 npm run collect.
